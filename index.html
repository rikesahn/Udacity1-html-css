First lessons in Programming and HTML

Preliminary remarks about learning how to code

When learning a new skill, thus going from novice to expert, you will go through several stages:

  1. ignorance
  2. awareness
  3. ability
  4. fluency

Getting from ignorance to awareness and ability takes minutes and days. Getting from ability to fluency might take years. Thus, it's important to have strategies to prevent frustration from turning into disappointment and demotivation:

  • ask for help (Google, online forums,...)
  • take a break
  • just keep on going, accept a temporary lack of complete understanding

5 ways of thinking that programmers need

  1. procedural thinking: Instructions that a computer can follow much faster than humans
  2. abstract thinking: finding generality amongst seemingly different things
  3. systematic debugging:
    1. Collect evidence
    2. generate theories
    3. test those theories
    4. fix problem
  4. technological empathy: Ability to understand how computers work. Computers are machines. But they cannot deal with ambiguous, context-related info ( at least not yet)
  5. systems thinking: Break down a big problem into smaller pieces

Lesson 1: Basics of the Web and HTML

The World Wide Web

  • the internet is the world's largest computer network
  • it consists of servers that contain HTML documents
  • you can access and display these files by using a program called browser
  • Hyperlinks connect to other documents
  • the main protocol of the web is called http and is used by servers to communicate with each other an with your browser
  • HTML documentation including tags can be looked up here

Lesson 2: creating a structured document

The different files that compose a website are comparable to the components of a house:
  • HTML files are like the different rooms
  • CSS files are like the style (colors, materials used,...)
  • JavaScript files are interactive components, similar to the TV remote
  • There are rules for how these structures can interact

HTML structure

  • HTML sites are built in a tree-like structure that is branching out
  • The elements in the tree are the HTML tags
  • Not all elements of the page's html-structure are visible in the browser
  • All elements are rectangular / box-shaped
  • this "boxified" design makes it easy to rearrange content quickly
  • "class attributes" are like labels on boxes that categorize them

Interplay of HTML, CSS and DOM

  • HTML (HyperText Markup Language) is the standard markup language used to create webpages and control their structure
  • CSS (Cascading Style Sheets) is a style sheet language used for describing the look and formatting of a document written in a markup language
  • DOM (document object model) refers to the tree-like structure of a page. It's a cross-platform and language-independent convention for representing and interacting with objects in HTML (and other markup languages)
